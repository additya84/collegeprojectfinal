Polymorphism in C++

Learn Polymorphism in C++ including Function Overloading, Operator Overloading and Runtime Polymorphism with examples.

📘 What is Polymorphism?

Polymorphism means "One Name, Many Forms". It allows the same function or operator to perform different tasks.

Example: A person can be a Student, Employee and Customer at the same time.

📚 Types of Polymorphism

  • Compile Time Polymorphism
  • Run Time Polymorphism

⚡ Compile Time Polymorphism

Compile Time Polymorphism is achieved using Function Overloading and Operator Overloading.

➕ Function Overloading

Defining multiple functions with the same name but different parameters. 

#include <iostream>
using namespace std;

class Print
{
public:

    void show(int x)
    {
        cout << "Integer: " << x << endl;
    }

    void show(double y)
    {
        cout << "Double: " << y << endl;
    }

    void show(string z)
    {
        cout << "String: " << z << endl;
    }
};

int main()
{
    Print p;

    p.show(10);
    p.show(15.5);
    p.show("BCA Hub");

    return 0;
}

Integer: 10
Double: 15.5
String: BCA Hub

➗ Operator Overloading

Operator Overloading means giving additional meaning to an operator. 

class Test
{
public:

    int num;

    Test operator +(Test t)
    {
        Test temp;

        temp.num = num + t.num;

        return temp;
    }
};

🚀 Run Time Polymorphism

Runtime Polymorphism is achieved using Function Overriding and Virtual Functions.

🔁 Function Overriding

#include <iostream>
using namespace std;

class Animal
{
public:
    virtual void sound()
    {
        cout << "Animal Sound";
    }
};

class Dog : public Animal
{
public:
    void sound()
    {
        cout << "Dog Barks";
    }
};

int main()
{
    Animal *a;

    Dog d;

    a = &d;

    a->sound();

    return 0;
}
Output: Dog Barks

🎯 Virtual Function

A virtual function is a member function that can be overridden in derived classes.

Virtual keyword enables Runtime Polymorphism in C++.

✅ Advantages of Polymorphism

  • Code Reusability
  • Flexibility
  • Easy Maintenance
  • Better Code Organization
